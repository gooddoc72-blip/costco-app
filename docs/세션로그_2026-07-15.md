## B. 카페24 → 사용자 스토어 대행 등록

관리자가 대상 사용자를 고르고 공용 카페24 카탈로그에서 상품을 불러와 **그 사용자의 네이버 스토어**에 대행 등록.

### 흐름 (건별 루프, `cafe24_page.py`)
0. `cafe24_api.get_product` → 상세 + 대표이미지(`detail_image`||`list_image`)
1. **📷 AI 제품사진 분석**(`af55596`): 대표이미지 bytes → `ai_service.analyze_product_photo` → `{name, origin, brand}`
   - `_fetch_image_bytes(url)` 헬퍼로 URL→bytes+media_type
   - 분석 상품명(`_base_name`)이 이후 전 단계의 기준
2. **카테고리 자동판단**: `naver_shopping_search(_base_name)` → `suggest_naver_category` → `search_naver_categories`로 leaf 확정. 폴백: AI가 준 `category` 키워드
3. **연관키워드 상품명 최적화**(`41b514e`): `naver_api.keyword_optimized_name(ad_key, ad_sec, cust, seed=_base_name, ai_key, category)` — 저경쟁(총검색량 100~300, 경쟁도 낮은순) 1~2개 + 대표어(최고검색량) → AI 조합
   - **상품명 하드 가드**: 최종명 비면 `_cf_name → _name → _base_name`
4. **태그ID**(`41b514e`): `build_seller_tags`(사전등록 태그만, ad_creds로 검색량순)
5. **속성**: AI 브랜드/원산지 우선, 없으면 카페24 `manufacturer_name`/`brand_name`/`origin_place_value`/`model_name`
6. **상세페이지**(`bf987b0`,`855afff`): 최상단 **상품명 블록**(중앙·굵게, HTML escape) + 🖼이미지 등록 옵션(`_build_image_detail`: 카페24 description의 `<img>` + 대표이미지 → 네이버 CDN `<img>` 스택)
7. `naver_api.register_product` — sale_price=마진공식, seller_code=카페24 상품번호

### UI
- 대상 사용자 selectbox + 마진율, 카페24 상품명 검색, 체크리스트(최대 30)
- **전체 선택 토글**(`b07b652`): 변화 감지 시 개별 `ag_ck_*` 세션값을 위젯 생성 전에 일괄 설정
- 토글: 🖼 상세이미지 등록 / 📷 AI 제품사진 분석
- 결과표(상품/카테고리/태그수/판매가/상태)

### naver_api 확장 (`41b514e`)
- `keyword_optimized_name` 신설(`naver_api/keywords.py`) + `__init__` 노출
- `register_product`에 `model_name`/`origin_content` 파라미터 추가(하위호환)

## C. 카페24 ↔ 코스트코 매칭 & 동기화 (`bf987b0`)

카페24 전 상품에 코스트코 번호를 매칭하고, 코스트코 상태를 카페24에 반영.

### 매칭 UI (자동추천 후 확인)
- "📥 카페24 전체 불러오기 + 자동매칭" → `get_all_products`(전상품) + `get_shared_products`
- 이름 유사도(`utils.calc_match_score`)로 코스트코번호 자동 추천, 이미 `custom_product_code` 있으면 점수 999
- `st.data_editor`에서 코스트코번호 열 수정 → **카페24 자체상품코드**(custom_product_code)에 저장
- 필터(전체/미매칭만/매칭만), 보이는 행만 저장·동기화

### 동기화 동작
- 코드 매칭분 → **코스트코 현재가 → 카페24 매입가(supply_price)**
- **코스트코 품절/판매종료 → 카페24 판매중지(selling=F)**
- 상태 판정: `costco_crawler.fetch_costco_status(no)` — 상세 API로 `{exists, available, price, reason}`. 404/삭제=판매종료, outOfStock/purchasable=False=품절, 네트워크 불확실=exists None→건너뜀(오탐 방지)

### cafe24_api 확장
- `update_custom_product_code` / `update_supply_price` / `update_selling_status(selling, display)`
- `get_all_products` 반환에 `custom_product_code`·`supply_price` 추가
