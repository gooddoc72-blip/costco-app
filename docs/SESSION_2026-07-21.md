### B. Next.js 이전 (수익계산만) — 사용자 결정됨
- 이유: Streamlit의 "위젯값이 화면에 없으면 폐기"(페이지 넘기면 박스비 리셋) 등 상태형 표에 부적합. 전체가 아닌 **수익계산 1페이지만** React+API로.
- **Phase1 완료**: `compute.py` (순수 함수, 패리티 0). **Phase1.5 완료**: 현 페이지가 compute.py 사용.
- **Phase2(다음)**: FastAPI로 `compute.py` + 조회/저장/삭제/영수증 API 노출.
- **Phase3**: React(Next.js) 수익계산 표. **Phase4**: nginx 라우팅·세션인증·컷오버.
- ⚠️ 돈 계산이라 매 변경마다 패리티 테스트(scratchpad `parity_compute.py`)로 불변 확인.
