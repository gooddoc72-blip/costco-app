## 2. 로컬 PC 설치형 + 1-PC 사용 인증 (핵심 작업)

웹(cocobiz.shop)은 그대로 유지하고, **각 PC에 설치해 본인 API키로 단독 실행 + 1키=1PC 인증**하는 설치판 추가.
결정: **설치 프로그램(Inno Setup) + 서버 온라인 검증 + 웹 유지 + 라이선스키=계정 자동로그인**.

### Phase 1 — 서버 라이선스 인프라 (a4f3225)
- **`db_license.py`**: `licenses` 테이블(auth.db) + 발급/검증/철회/PC해제/삭제
  - `verify_and_bind(key, machine_id)`: 최초엔 그 PC에 바인딩, 이후 동일 PC만 허용 (1키=1PC)
  - 반환 code: `ok | activated | invalid_key | revoked | machine_mismatch`
- **`license_api.py`**: stdlib HTTP 검증 엔드포인트 (Flask 불필요)
  - `GET /license/ping`, `POST /license/verify {key, machine_id}`
- **관리자 페이지**(admin_page.py): 🔑 로컬 설치형 라이선스 발급/조회/정지/PC해제/삭제 UI
- **서버 인프라**(SSH로 설정, repo 외부):
  - systemd 서비스 `costco-license` (127.0.0.1:8600, `/opt/costco-app/venv` python)
  - nginx `/license/` → 127.0.0.1:8600 프록시 (백업: `/etc/nginx/sites-enabled/costco-app.bak0625`)
  - HTTPS 실검증 완료: activated→ok→machine_mismatch 정상

### Phase 2 — 클라이언트 인증 게이트 (c9f1b5d, 886c5b2)
- **`license_client.py`**: `is_local_mode()`(env `COSTCO_LOCAL=1`), `get_machine_id()`(Windows MachineGuid),
  `get_stored_key/save_key`(`~/.costcobiz_license.json`), `verify_license()`(**stdlib urllib**, requests 의존 제거)
- **`app.py`**: 로컬 모드일 때만 시작 시 검증 → 미인증이면 활성화 화면(키 입력→서버검증→저장), 인증되면 진입. 웹 모드는 게이트 건너뜀.
- 이 PC MachineGuid(`51f0bc78-…`)로 실검증 완료.

### Phase 3 — 빌드/설치 자산 (bec99b5, 395aaaf) + 실빌드 검증
- **`local_build/run_costco.bat`**: `COSTCO_LOCAL=1` + Streamlit 실행 런처
- **`local_build/build.bat`**: 포터블 Python(3.11) + 의존성 + 코드 번들 → `_build/`
- **`local_build/installer.iss`**: Inno Setup 설치 프로그램 정의
- **`local_build/README.md`**: 빌드/설치/라이선스 발급 가이드
- `.gitignore`: `local_build/_build/`, `.costcobiz_license.json` 제외
- **실빌드 검증 완료**: 포터블 Python 3.11.9 + 의존성 전체 설치 + 핵심 모듈 import + MachineGuid + 라이선스 검증 OK.
- **남은 수동 단계**: 빌드 PC에서 `build.bat` → Inno Setup으로 `installer.iss` 컴파일 → `CocoBiz_Setup.exe`.

### 회원/로그인 방식 — 라이선스키=계정 자동로그인 (15192cb)
- **`db_auth.ensure_local_user(username, display_name)`**: 라이선스 계정을 로컬 auth.db에 자동 생성(active),
  username은 영문/숫자만 정제(DB 파일명 안전), 멱등.
- **`db_license.verify_and_bind`**: 표시이름(`display`=memo) 추가 반환.
- **`app.py`**: 로컬 모드 활성화 성공 시 라이선스 계정으로 **자동 로그인** (회원가입/비번 불필요).
- **관리자 발급 UI**: 계정 사용자명(영문) / 표시이름(가게명) 분리 입력.
- 흐름: 관리자가 🔑 키 발급(계정명 지정) → 사용자 첫 실행 시 키 입력 → 그 PC 바인딩 + 그 계정으로 자동 로그인 → 본인 API키 설정 후 사용.
