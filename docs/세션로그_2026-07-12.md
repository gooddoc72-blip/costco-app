## A. 네이버 상품 등록(건별) 고도화

### 1. AI 연관태그 자동 생성·등록 (`ac0679f`)
- **커머스 API 태그 3종 연동** (`naver_api/products.py`)
  - `search_recommend_tags` — `GET /external/v2/tags/recommend-tags?keyword=` → `[{code,text}]` (사전 등록 태그만 code 부여)
  - `filter_restricted_tags` — `GET /external/v2/tags/restricted-tags?tags=` (배열 배치) → 제한 태그 제거
  - `build_seller_tags` — AI 후보 20~25개 → 추천태그 검증 → 제한태그 제거 → (검색광고 검색량×관련도) 상위 10개
    - `get_token` 1회만 발급해 재사용(bcrypt 비용), rate limit 스로틀
- `register_product`: `seller_tags` → `originProduct.detailAttribute.seoInfo.sellerTags` 주입. **태그 거부 시 태그 빼고 1회 재시도**(등록 자체는 반드시 성공)
- UI(`naver_register_page.py`): 건별 흐름에 "🤖 AI 태그 10개 생성" → `data_editor` 검토(체크해제/행추가/태그ID·월검색량) → 등록
- **가격수정 PUT의 sellerTags 스트립은 유지** (등록용/수정용 분리)

### 2. 이미지 1000×1000 자동 변환 (`476d48e`, `7f82821`, `d2a1731`)
- `upload_product_image` 진입부에서 정사각 변환 → **모든 등록 흐름(건별·일괄·카페24·재등록) 공통 적용**

- "🔲 1000×1000 변환 미리보기" 체크박스 + 등록 성공 시 실제 CDN 대표이미지 표시

### 4. 상세페이지 제품 설명 (`d2a1731`, `61c7886`, `1ddd137`, `aa0e0ac`)

- **문장별 줄바꿈 + 문장 사이 빈 줄**(`_desc_to_lines`, `\n\n` → `<br><br>`), 가운데 정렬

- **상세페이지 "제품 상세정보" 표 자동 삽입** (`_food_info_html`, 등록 실패 위험 0 — HTML만)
- **상품정보제공고시 FOOD** 자동입력 (`_build_food_notice` → `productInfoProvidedNotice`):
  - `foodItem/producer/weight/amount/packDate/expirationDate/productComposition/keep/adCaution`
  - 미상은 "상품 상세페이지 참조", 유통기한·제조일=상세참조, 원산지=상세참조(사용자 결정)
  - **FOOD 거부 시 ETC 자동 복원 후 재등록** + 거부 원인 표시
  - ⚠️ FOOD 필드명은 인증문서로 100% 확정 못함 → 실등록 1건으로 검증 필요(미완)
